# 语音通话功能使用说明

## 功能概述

为AI角色聊天系统增加了实时语音通话功能，支持：
- 实时语音对话
- 音频可视化显示
- 静音/取消静音控制
- 扬声器开关
- 屏幕共享（预留功能）

## 布局设计

### 1. 语音通话按钮位置

**头部右侧**（主要入口）：
- 位于聊天界面头部右侧
- 显示在线状态旁边
- 悬停显示提示文字
- 通话中显示音频波形

**输入区右侧**（快捷入口）：
- 位于输入框底部发送按钮旁
- 点击后自动滚动到头部按钮
- 方便用户快速发起通话

### 2. 通话模态框

点击语音按钮后弹出通话模态框，包含：
- 角色信息和连接状态
- 实时音频可视化
- 通话控制按钮
- 通话信息显示
- 使用提示

## 使用流程

### 发起通话
1. 点击头部或输入区的语音通话按钮
2. 等待WebRTC连接建立（约1秒）
3. 连接成功后开始实时语音对话

### 通话中控制
- **静音按钮**：关闭/开启麦克风
- **扬声器按钮**：切换音频输出
- **屏幕共享**：预留功能（待开发）
- **挂断按钮**：结束通话

### 通话特性
- 实时音频双向传输
- 音频电平可视化
- 通话时长显示
- 网络质量监控

## 技术特性

### 1. 免费方案
- **信令服务器**：FastAPI + WebSocket
- **音频传输**：WebRTC P2P直连
- **TTS服务**：Edge TTS（微软官方免费）
- **STUN服务**：公共免费服务器

### 2. 性能优化
- 使用WebRTC Opus编解码器
- 音频流低延迟传输
- CSS硬件加速动画
- 响应式布局适配

### 3. 用户体验
- 毛玻璃效果设计
- 平滑过渡动画
- 触摸友好界面
- 无障碍支持

## 响应式设计

- **桌面端**：双入口设计，头部更显眼
- **平板端**：调整按钮大小和位置
- **手机端**：仅显示头部按钮，隐藏输入区按钮
- **暗色模式**：自动适配系统主题

## 注意事项

1. **浏览器要求**：
   - 支持WebRTC的现代浏览器
   - 需要HTTPS连接（本地开发除外）

2. **网络要求**：
   - 稳定的网络连接
   - 建议使用耳机减少回声

3. **隐私说明**：
   - 通话内容不会记录
   - 音频仅在P2P传输中处理

## 后续优化

1. **TTS切换**：支持Edge TTS/Coqui TTS切换
2. **语音识别**：可选集成语音转文本
3. **背景音乐**：添加背景音乐选项
4. **音效**：消息提示音效

## 故障排除

### 连接失败
- 检查网络连接
- 尝试刷新页面
- 检查浏览器权限

### 无声音频
- 检查麦克风权限
- 尝试切换输出设备
- 检查系统音量设置

### 延迟过高
- 检查网络延迟
- 尝试更换STUN服务器
- 关闭其他占用带宽的应用
